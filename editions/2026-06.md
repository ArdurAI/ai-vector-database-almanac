## Vector Databases — The Parser Is the Bottleneck

### The state of the category

Market: $2.65B (2025) → $8.44B (2030) at 23-27.5% CAGR. 25+ distinct vector databases actively maintained. The "Postgres vs. dedicated vector DB" debate is the central infrastructure fork. Document parsing has emerged as the #1 RAG quality bottleneck — no vector DB can retrieve what wasn't properly parsed in the first place.

The 2026 vector database landscape is characterized by **convergence on features** and **divergence on operational models**:
- Every major vector DB now claims hybrid search, metadata filtering, and binary quantization.
- The real differentiator is not feature count but **ops burden**, **cost curve**, and **deployment flexibility**.
- Self-hosted open-source (Qdrant, Weaviate, Milvus) vs. managed proprietary (Pinecone, Zilliz, Turbopuffer) is the structural divide.

## Findings

### Indexing & Algorithms
- **Hybrid search is now default**: Pure vector retrieval is a starting point; production RAG requires BM25 + dense + reranking. Weaviate leads on native hybrid; Pinecone added sparse-dense Q1 2026 but remains less mature.
- **New indexing algorithms beyond HNSW**: ACORN (Qdrant filtered HNSW), RaBitQ (FAISS), SOAR (ScaNN), StreamingDiskANN (pgvectorscale), Vortex (Zilliz), CAGRA (Milvus GPU). The HNSW monoculture is ending.
- **Binary quantization is table stakes**: Qdrant (32×), Weaviate (32×), Milvus/Zilliz all offer BQ. Reduces memory 10-30× at minimal recall loss. The cost story of vector DBs in 2026 is largely a BQ story.
- **GPU acceleration is niche but growing**: Milvus leads with CAGRA (50× speedup claims). Qdrant v1.12+ will add GPU. Weaviate exploring RAPIDS. GPU-first indexing may dominate 2027 for billion-scale workloads.

### Operational Models
- **Cost tipping point at 60-80M queries/month**: Self-hosted Qdrant/Weaviate undercuts Pinecone Serverless by 3-10× above this threshold. The $300/month Pinecone bill is the migration trigger to self-hosted.
- **Postgres is the default for 90% of products**: pgvector under 10M vectors is the "just use Postgres" answer. pgvectorscale extends this to 50M+. The dedicated vs. Postgres debate is scale-dependent, not religious.
- **Managed services converge on serverless**: Pinecone, Qdrant Cloud, Weaviate Cloud, Zilliz Cloud all offer pay-as-you-go tiers. The differentiator is not serverless vs. provisioned but cost transparency and cold-start behavior.
- **Cold starts remain a managed-service problem**: Pinecone Serverless: 200-800ms after inactivity. Turbopuffer claims no cold starts. Vorkath claims no cold starts. Self-hosted has no cold starts by definition.

### RAG Infrastructure Shift
- **Agentic RAG shift**: Vector DBs evolving from passive retrieval to active knowledge engines — Pinecone Nexus, Chroma Context-1, Weaviate Engram. This is not marketing; it changes the API surface from "search" to "reason."
- **MCP servers arrive**: Weaviate v1.37 (April 2026) shipped a built-in MCP server at `/v1/mcp`. This removes the custom integration layer between LLMs and vector DBs that every agentic system has required since 2024. Other vendors will follow.
- **Document parsing is the #1 bottleneck**: The vector DB retrieves vectors; the parser determines what vectors mean. Firecrawl, Docling, LlamaParse, and Unstructured are as critical as the vector DB itself. Parser quality variation is 2-5× more impactful on RAG accuracy than vector DB choice.
- **Rerankers are now required**: Cohere Rerank v3.5, Voyage AI Rerank-2.5, BGE Reranker v2. First-stage retrieval (vector DB) + second-stage reranking is the production RAG standard. No vector DB alone is sufficient.

### What surfaced
- **The " 2" file problem**: The almanac was bootstrapped from `ai-llmops-almanac` template, leaving duplicate files. Cleaned before first public commit.
- **roster.json scope creep**: The roster includes RAG frameworks, document parsers, and rerankers — not just vector DBs. This is intentional (the almanac covers RAG infrastructure), but may need sub-category breakdown in future editions.
- **No real benchmarks yet**: Harness uses synthetic random data. Real ANN-Benchmarks datasets (SIFT1M, GloVe-100, Deep1B) must be downloaded to `benchmarks/data/` before meaningful performance numbers can be produced.
- **Adapter contract frozen**: The `VectorDBAdapter` base class is now the canonical interface. All future adapters must implement `setup()`, `load()`, `build_index()`, `await_ready()`, `search()`, `delete()`, `teardown()`. The `await_ready()` barrier is where async designs get their cost measured.
- **Data quality**: Market size figures ($2.65B → $8.44B) are from analyst reports and should be treated as directional, not precise. Star counts and pricing are from vendor pages and third-party analysis as of June 2026.
